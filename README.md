## Разработка грамматики

### Формальное описание грамматики G[Z]

G[Z] = (Vt, Vn, P, Z)

### Множество терминальных символов

Vt = { String, id, =, string_const, ;, sp, +, -, /, *, (, ) }

### Множество нетерминальных символов

Vn = { Z, A, B, C, D }

### Правила вывода

| № | Правило | Описание |
|---|---------|----------|
| 1 | Z → String A | Объявление строковой переменной |
| 2 | A → id B | Идентификатор |
| 3 | B → = C | Оператор присваивания |
| 4 | C → string_literal D | Строковая константа |
| 5 | D → ; | Конец оператора |

### Расшифровка терминалов

| Терминал | Описание |
|----------|----------|
| String | Ключевое слово String |
| id | Идентификатор (буква, затем буквы/цифры/подчеркивание) |
| = | Оператор присваивания |
| string_literal | Строковая константа в двойных кавычках |
| ; | Точка с запятой |
| sp | Пробел, табуляция |

### Альтернативная запись (БНФ)


<программа> ::= <объявление_строки>
<объявление_строки> ::= "String" <идентификатор> "=" <строковая_константа> ";"
<идентификатор> ::= <буква> (<буква> | <цифра> | "_")*
<строковая_константа> ::= '"' <последовательность_символов> '"'
<последовательность_символов> ::= <любой_символ>*
<любой_символ> ::= [\u0000-\uFFFF] кроме '"'

## Классификация грамматики (по Хомскому)

Разработанная грамматика является **контекстно-свободной**, то есть относится к **грамматикам типа 2** по классификации Хомского.

### Обоснование

Каждое правило вывода имеет вид: A → α

где:
- **A** — один нетерминальный символ;
- **α** — последовательность терминальных и нетерминальных символов.

Так как в левой части каждого правила стоит ровно один нетерминал, разработанная грамматика является контекстно-свободной и допускает построение синтаксического анализатора методом рекурсивного спуска.

---

## Метод анализа

Так как разработанная грамматика является контекстно-свободной, для синтаксического анализа используется **метод рекурсивного спуска**.

### Основная идея

Основная идея метода рекурсивного спуска состоит в том, что каждому нетерминалу грамматики ставится в соответствие отдельная процедура синтаксического анализа. Эти процедуры вызываются в соответствии с правилами грамматики и последовательно распознают корректную цепочку лексем.

### Программные процедуры

В программе реализованы следующие основные процедуры:

| Процедура | Соответствующий нетерминал | Описание |
|-----------|---------------------------|----------|
| `parseStringDeclaration()` | Z | Главная процедура анализа |

## Диагностика и нейтрализация синтаксических ошибок (метод Айронса)

### Принцип работы метода Айронса

При обнаружении синтаксической ошибки анализатор:
1. Фиксирует позицию ошибки (строка, столбец)
2. Запоминает ошибочный фрагмент
3. Записывает описание ошибки
4. Пропускает символы до следующего синхронизирующего маркера
5. Продолжает анализ с начала синтаксической конструкции

### Синхронизирующие маркеры

Для метода Айронса определены следующие синхронизирующие маркеры:
- `;` — точка с запятой (конец оператора)
- `String` — ключевое слово (начало нового объявления)
- Конец строки

### Типы обнаруживаемых ошибок

| Код | Ошибочный фрагмент | Описание ошибки | Способ нейтрализации |
|-----|-------------------|-----------------|---------------------|
| E001 | (любой символ) | Отсутствует ключевое слово "String" | Пропуск до следующего возможного начала |
| E002 | (не id) | Ожидался идентификатор | Пропуск до '=' или '"' |
| E003 | (не =) | Ожидался оператор присваивания "=" | Пропуск до '"' |
| E004 | (не ") | Ожидалось открытие строковой константы | Вставка '"' |
| E005 | \n или конец | Незакрытая строковая константа | Добавление '"' |
| E006 | (не ;) | Ожидалась точка с запятой ";" | Вставка ';' |
| E007 | (неправильный id) | Недопустимый символ в идентификаторе | Пропуск символа |
| E008 | (пустая строка) | Отсутствует объявление String | Ожидание ввода |
| E009 | (недопустимые символы) | Недопустимые символы (&&, $$ и т.д.) | Группировка в одну ошибку |

### Алгоритм нейтрализации (метод Айронса)


procedure Synchronize():
syncSet = {CODE_SEMICOLON, CODE_KEYWORD}

while currentPos < tokens.length:
current = tokens[currentPos]
if current.code in syncSet:
if current.code == CODE_SEMICOLON:
currentPos++
break
addError(current, "Недопустимый символ, пропущен")
currentPos++
end while
end procedure

## Навигация по ошибкам

В программе реализована удобная навигация по ошибкам:
- **Двойной клик** на строке с ошибкой в таблице "Ошибки" автоматически перемещает курсор в область редактирования на позицию ошибочного фрагмента
- Ошибочный фрагмент **выделяется** цветом для наглядности
- Строка с ошибкой **прокручивается** в видимую область
- В статусной строке отображается информация о выбранной ошибке
