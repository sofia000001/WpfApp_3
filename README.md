## Классификация грамматики (по Хомскому)

Согласно классификации Хомского, грамматика G[‹START›] является **автоматной**, то есть относится к **грамматикам типа 3**.

### Обоснование

Все правила вывода имеют вид:

- **A → aB** (праволинейная форма)
- **A → a** (терминальная форма)

где:
- **A, B** — нетерминальные символы;
- **a** — терминальный символ.

**Примеры продукций:**

| Тип | Правило |
|-----|---------|
| A → aB | ‹START› → String ‹SPACE› |
| A → aB | ‹SPACE› → sp ‹ID› |
| A → aB | ‹ID› → letter ‹IDREM› |
| A → aB | ‹IDREM› → letter ‹IDREM› |
| A → aB | ‹IDREM› → digit ‹IDREM› |
| A → aB | ‹IDREM› → = ‹ASSIGN› |
| A → aB | ‹ASSIGN› → " ‹STRING› |
| A → aB | ‹STRING› → char ‹STRINGREM› |
| A → aB | ‹STRINGREM› → char ‹STRINGREM› |
| A → aB | ‹STRINGREM› → " ‹END› |
| A → a  | ‹END› → ; |

Все продукции являются **праволинейными**, левая рекурсия отсутствует.

Следовательно, грамматика является **полностью автоматной** и допускает построение конечного автомата для распознавания цепочек.

## Диагностика и нейтрализация синтаксических ошибок (метод Айронса)

### Принцип работы метода Айронса

При синтаксическом анализе входной строки возможно возникновение ошибок, связанных с нарушением структуры конструкции. Для их обработки используется метод Айронса.

Пусть входная цепочка имеет вид: Tt

где:
- **T** — текущий (ошибочный) символ;
- **t** — оставшаяся часть входной цепочки.

**Суть метода Айронса заключается в следующем:**

1. В процессе разбора определяется текущий недостроенный куст дерева разбора
2. Формируется множество допустимых символов, которые могут следовать в данной позиции
3. Если текущий символ не принадлежит этому множеству, он считается ошибочным
4. Выполняется последовательное удаление символов входной цепочки до тех пор, пока текущий символ не станет допустимым
5. После этого синтаксический анализ продолжается

### Особенности для автоматной грамматики

Для автоматной грамматики характерно, что в любой момент времени существует **только один недостроенный куст**, поэтому алгоритм упрощается:

- Ошибка всегда относится к текущему состоянию автомата
- Восстановление выполняется путем пропуска входных символов до допустимого

### Таблица обнаруживаемых ошибок

| Код | Ошибочный фрагмент | Описание ошибки | Способ нейтрализации |
|-----|-------------------|-----------------|---------------------|
| E001 | (не String) | Ожидалось ключевое слово String | Пропуск символов до буквы 'S' |
| E002 | (не идентификатор) | Ожидался идентификатор | Пропуск до символа '=' |
| E003 | (не =) | Ожидался оператор присваивания | Пропуск до '=' |
| E004 | (не ") | Ожидалось начало строковой константы | Вставка или пропуск до '"' |
| E005 | перевод строки или EOF | Незакрытая строковая константа | Добавление закрывающей '"' |
| E006 | (не ;) | Ожидалась точка с запятой | Вставка ';' |
| E007 | недопустимые символы | Обнаружены лишние символы | Пропуск до допустимого символа |
| E008 | лишние пробелы/символы | Нарушение структуры конструкции | Игнорирование лишних символов |
