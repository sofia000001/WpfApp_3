## Разработка грамматики

### Формальное описание грамматики G[Z]

G[Z] = (Vt, Vn, P, Z)

### Множество терминальных символов

Vt = { S, t, r, i, n, g, sp, id, =, ", char, ;, +, -, /, *, (, ), num }

где:
- **S** — буква 'S'
- **t** — буква 't'
- **r** — буква 'r'
- **i** — буква 'i'
- **n** — буква 'n'
- **g** — буква 'g'
- **sp** — пробельный символ
- **id** — идентификатор (буква, затем буквы/цифры/подчеркивание)
- **=** — оператор присваивания
- **"** — двойная кавычка
- **char** — любой символ (кроме " и \n)
- **;** — точка с запятой
- **+** — оператор сложения
- **-** — оператор вычитания
- **/** — оператор деления
- *** — оператор умножения
- **(** — открывающая скобка
- **)** — закрывающая скобка
- **num** — цифра

### Множество нетерминальных символов

Vn = { Z, A, B, C, D, E, F, G, H, I, J, K, L, M, N, O }

### Правила вывода (автоматная грамматика)

Автоматная грамматика — это грамматика типа 3 по классификации Хомского, где все правила имеют вид:
- **A → aB** (праволинейная форма)
- **A → a** (терминальная форма)

где A, B — нетерминалы, a — терминал.


### Альтернативная запись в нотации БНФ:


<программа> ::= "S" <A>
<A> ::= "t" <B>
<B> ::= "r" <C>
<C> ::= "i" <D>
<D> ::= "n" <E>
<E> ::= "g" <F>
<F> ::= <sp> <G>
<G> ::= <id> <H>
<H> ::= <sp> <I>
<I> ::= "=" <J>
<J> ::= <sp> <K>
<K> ::= '"' <L>
<L> ::= '"' <N> | <char> <M>
<M> ::= '"' <N> | <char> <M>
<N> ::= <sp> <O>
<O> ::= ";"

## Классификация грамматики (по Хомскому)

Разработанная грамматика является **автоматной**, то есть относится к **грамматикам типа 3** по классификации Хомского.


Все правила вывода имеют вид:
- **A → aB** (праволинейная форма)
- **A → a** (терминальная форма)

где:
- **A, B** — нетерминальные символы;
- **a** — терминальный символ.

---

## Метод анализа

Так как разработанная грамматика является автоматной (тип 3), для синтаксического анализа используется **метод конечного автомата**.


### Алгоритм работы конечного автомата

1. Начальное состояние: state = START (0)
2. Пока есть входные символы:
3. Считать следующий символ c
4. Определить переход из текущего состояния по символу c
5. Если переход определен:
5.1. Перейти в новое состояние
5.2. Если новое состояние является терминальным: 
    - Выдать код лексемы
    - Сбросить состояние в START (0)

6. Если переход не определен:
6.1. Выдать код ошибки (ERROR)
6.2. Сбросить состояние в START (0)

## Диагностика и нейтрализация синтаксических ошибок (метод Айронса)

### Принцип работы метода Айронса

При обнаружении синтаксической ошибки анализатор:
1. Фиксирует позицию ошибки (строка, столбец)
2. Запоминает ошибочный фрагмент
3. Записывает описание ошибки
4. Пропускает символы до следующего синхронизирующего маркера
5. Продолжает анализ с начала синтаксической конструкции


### Таблица обнаруживаемых ошибок

| Код | Ошибочный фрагмент | Описание ошибки | Способ нейтрализации |
|-----|-------------------|-----------------|---------------------|
| E001 | (любой символ) | Отсутствует ключевое слово "String" | Пропуск до следующего возможного начала |
| E002 | (не id) | Ожидался идентификатор | Пропуск до '=' или '"' |
| E003 | (не =) | Ожидался оператор присваивания "=" | Пропуск до '"' |
| E004 | (не ") | Ожидалось открытие строковой константы | Вставка '"' |
| E005 | \n или конец | Незакрытая строковая константа | Добавление '"' |
| E006 | (не ;) | Ожидалась точка с запятой ";" | Вставка ';' |
| E007 | (недопустимые символы) | Недопустимые символы (&&, $$ и т.д.) | Группировка в одну ошибку |
